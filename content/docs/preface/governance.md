# 治理

## 角色和职责

### 作者

Harminder Virk（AdonisJS 的创建者）担任项目作者。项目作者负责项目的治理、标准和方向。总结如下：

- 项目作者决定哪些新项目应该存在于 AdonisJS 保护伞下。
- 项目作者负责为项目分配负责人，并在现有负责人卸任时将项目转让给新的负责人。
- 作者有责任分享/记录框架的愿景，并使项目负责人与之保持同步。

### 项目负责人

AdonisJS 是由核心团队创建和管理的多个包的组合。所有这些包都由项目作者选定的项目负责人领导。

在几乎所有情况下，包的创建者都担任项目负责人，因为他们是投入最初努力将想法变为现实的人。

项目负责人在项目的所有决策方面拥有最终决定权。然而，由于社区总是有能力分叉，这个人完全对社区负责。项目负责人有责任为项目设定战略目标，并清楚地向社区传达这些目标。他们还必须理解整个社区，并努力满足尽可能多的冲突需求，同时确保项目长期生存。

在许多方面，项目负责人的角色是关于外交。关键是确保随着项目的扩展，合适的人被赋予对它的影响力，社区团结在项目负责人的愿景背后。然后，负责人的工作是确保核心团队成员（见下文）代表项目做出正确的决定。一般来说，只要核心团队成员与项目战略保持一致，项目负责人将允许他们按期望进行。

:::note
项目负责人不能归档或决定从 AdonisJS 保护伞下移除项目。他们可以决定停止项目工作，在这种情况下，我们会找到新的项目负责人。
:::

### 核心团队

核心团队成员是已经对项目做出多次宝贵贡献的贡献者，现在被依赖来直接为仓库编写代码以及筛选他人的贡献。在许多情况下，他们是程序员，但也可能以不同角色做出贡献，例如社区参与。通常，核心团队成员将专注于项目的特定方面，并将带来赢得社区和项目负责人尊重的专业知识和理解水平。核心团队成员的角色不是官方职位，它只是社区中有影响力的成员在项目负责人向他们寻求指导和支持时会发现自己处于的位置。

核心团队成员对项目的整体方向没有权威。然而，他们确实拥有项目负责人的信任。核心团队成员的工作是确保负责人了解社区的需求和集体目标，并帮助开发或引出对项目的适当贡献。通常，核心团队成员被赋予对其特定责任领域的非正式控制权，并被分配直接修改源代码某些部分的权利。也就是说，尽管核心团队成员没有明确的决策权，但他们经常会发现他们的行为与负责人做出的决定同义。

#### 活跃的核心团队成员

活跃的核心团队成员定期为项目做出贡献。活跃的核心团队成员通常有一个或多个重点领域 - 在大多数情况下，他们将负责子项目仓库中的定期问题分类、错误修复、文档改进或功能开发。

#### 核心团队荣誉成员

一些过去做出过宝贵贡献的核心团队成员可能由于各种原因不再能够承诺相同水平的参与。这是完全正常的，对项目的任何过去贡献仍然非常感谢。这些核心团队成员因其贡献而被授予核心团队荣誉成员称号，并欢迎随时恢复积极参与。

### 贡献者

贡献者是社区成员，他们要么没有成为核心团队成员的愿望，要么还没有被项目负责人给予机会。他们做出宝贵的贡献，例如下面列出的那些，但通常没有权限直接更改项目代码。贡献者通过沟通工具与项目互动，例如 RFC 讨论、GitHub 议题和拉取请求、Discord 聊天室和论坛。

任何人都可以成为贡献者。对项目的承诺没有期望，没有特定的技能要求，也没有选择过程。要成为贡献者，社区成员只需执行一个或多个对项目有益的行动。

一些贡献者将作为用户与项目互动，但也会发现自己正在做以下一项或多项工作：

- 支持新用户（当前用户通常提供最有效的新用户支持）
- 报告错误
- 识别需求
- 编程
- 协助项目基础设施
- 修复错误
- 添加功能

随着贡献者获得经验和对项目的熟悉，他们可能会发现项目负责人开始越来越依赖他们。当这种情况开始发生时，他们逐渐承担核心团队成员的角色，如上所述。

### 用户

用户是有项目需求的社区成员。他们是社区中最重要的成员：没有他们，项目将没有目的。任何人都可以成为用户；没有特定要求。

应鼓励用户尽可能多地参与项目和社区的生活。用户贡献使项目团队能够确保他们满足这些用户的需求。常见的用户活动包括（但不限于）：

- 宣传项目。
- 从新用户的角度告知开发者项目的优势和劣势。
- 提供精神支持（一句"谢谢"大有帮助）。
- 通过 GitHub Sponsors 提供财务支持。

继续与项目及其社区互动的用户通常会发现他们变得越来越参与。这些用户可能会继续成为贡献者，如上所述。

## 支持

鼓励社区中的所有参与者为项目管理基础设施中的新用户提供支持。提供这种支持是为了发展社区。寻求支持的人应该认识到，项目中提供的所有支持活动都是自愿的，因此是在时间允许的情况下提供的。需要保证响应时间或结果的用户因此应寻求购买支持合同。然而，对于那些愿意根据项目条款与项目互动并愿意帮助支持其他用户的人来说，社区支持渠道是理想的选择。

### 货币捐赠

对于开放开发项目来说，金钱不如积极贡献重要。然而，有些人或组织资金充裕但时间匮乏，更愿意以现金形式做出贡献。如果你想进行重大捐赠，你也许可以赞助我们实现新功能或修复一些错误。项目网站提供了明确的指导如何进行捐赠。

如果你使用该项目作为创收产品的业务，赞助其开发是有商业意义的。它确保你所依赖的项目保持健康和积极维护。它还可以改善在我们社区的曝光度，并更容易吸引新开发者。

## 品牌和所有权

AdonisJS（拼写时末尾有"JS"）是 Harminder Virk 的注册商标。

只有 `@adonisjs` npm 范围内的项目和 AdonisJS GitHub 组织下的项目才由核心团队管理和官方支持。

此外，你不得以可能错误地暗示与 AdonisJS 有任何官方联系或认可的方式使用 AdonisJS 名称或徽标。以可能导致客户混淆的方式使用 AdonisJS 名称或徽标是不允许的。

这包括以强调 AdonisJS 品牌的方式命名产品或服务，如"AdonisJS UIKit"或"AdonisJS Studio"，以及在域名中如"adonisjs-studio.com"。

相反，你必须使用自己的品牌名称，以明确区分它与 AdonisJS。

此外，未经明确书面同意，你不得将我们的商标用于 T 恤、贴纸或其他商品。

## AdonisJS 保护伞下的项目

AdonisJS 保护伞下的项目是项目作者的知识产权。一旦由项目负责人创建的项目成为"AdonisJS GitHub 组织"的一部分，或者如果它是在 `@adonisjs` npm 范围下发布的，项目负责人就不能删除或放弃该项目。
